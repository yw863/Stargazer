# StarGazer — Impeccable Design Context

## Design Context

### Users

Urban astronomy enthusiasts in China (Yangtze River Delta), beginner-to-intermediate level. Comfortable with technical terminology (Bortle scale, apparent magnitude, altitude angle, elongation). They come to the tool with a specific intent — "Is tonight worth going?" — and need efficient, confident answers. They are operating an instrument — but a beautiful one, the kind you enjoy holding.

### Brand Personality

**Three words:** precise · elegant · expectant

The experience should feel like opening a well-made field notebook — one with a cloth cover, thread-bound spine, and a hand-drawn celestial chart on the inside front page. There is anticipation in the interface: the quiet before a clear night. Data is the hero — but the stage it stands on matters. A bare wall and a spotlight are not the same as an empty room.

### Reference & Anti-Reference

**Reference (aspire toward):**
- **Are.na aesthetic:** Make room breath, color that conveys information, linear / line-based decorative elements — elegant, intellectual, spare
- **Printed ephemeris & tide tables:** Dense tabular data typeset with care — column alignment, hairline rules, the beauty of repetition and precision. Numbers are the texture of the page.
- **Dieter Rams–era Braun:** Instrument panels where every dial, every mark, every scale line is considered. The beauty is *in* the functionality, not painted on top of it.
- **Classical star chart engravings:** Constellation lines, coordinate grids, fine serif labels in the margins — geometry and typography creating atmosphere without illustration.

**Anti-references (never look like these):**
- SkySafari-style apps: realistic star photography, glow effects, "space blue" gradients, neon-on-dark
- NASA public outreach: hero banners, exclamation headlines, full-bleed photography
- Social / community products: like counts, comment threads, avatar rows, engagement visual language
- Generic SaaS: white backgrounds, rounded card stacks, blue CTA buttons, dashboard-grid layouts
- **Equally important anti-reference — "dark mode nothing":** A near-black screen with only text on it is not design. If you strip away every decorative element and are left with plain text on a flat background, you have gone too far. Restraint means *the few things you keep are exquisite*, not that nothing is there.

### Aesthetic Direction

**Primary motif — Ephemeris table, elevated:**
Tabular star catalogue and almanac aesthetics — but treated as a design opportunity, not a limitation. Monospace numerals in fixed-width columns, light hairline dividers, data aligned as if typeset for a printed ephemeris. Numbers are the visual texture of the page. The interface should feel closer to a beautifully typeset JPL Horizons output or a collector's edition tide table than to a modern web app.

**What "ephemeris aesthetic" looks like in practice (positive examples):**
- A cluster of numeric readouts where font size, weight, and spacing create clear hierarchy — the predicted magnitude in large Geist Mono, supporting data smaller, a hairline rule underneath with generous but intentional padding
- Column-aligned data where the vertical rhythm of numbers creates its own visual pattern, like coordinates on a chart
- A single decorative hairline (0.5px, ~15% opacity of accent color) that runs horizontally to divide sections — not a border, a *breath*
- Subtle typographic events: a section label in small-caps Spectral tracking at 0.12em, a date rendered with deliberate letterspacing, a `·` separator that participates in the rhythm rather than just sitting between words
- Background that isn't dead flat: a very subtle radial gradient from `#080c14` at center to `#060a11` at edges, or an extremely faint grid pattern (opacity 2–4%) evoking graph paper or celestial coordinate lines

**Theme:** Deep dark. Background near-black with a blue-navy tint (`#080c14`). This is a product used at night. But the dark is not featureless — it has depth, like a clear night sky has depth. Achieve this through micro-variations in value, not through gradients or glow.

**Color palette:**

| Role | Value | Usage |
|------|-------|-------|
| Background (base) | `#080c14` | Primary surface |
| Background (elevated) | `#0d1220` | Cards, modals, elevated panels — use sparingly |
| Hairline / border | `rgba(91,141,217, 0.12)` | Decorative rules, section dividers, table lines |
| Text (primary) | `rgba(255,255,255, 0.87)` | Headings, key data |
| Text (secondary) | `rgba(255,255,255, 0.55)` | Labels, descriptions, supporting info |
| Text (tertiary) | `rgba(255,255,255, 0.32)` | Disabled states, placeholder, metadata |
| Accent | `#5b8dd9` | The single most important element on screen. Sparingly. |
| Accent (muted) | `rgba(91,141,217, 0.15)` | Hover states, subtle highlights, active indicators |
| Warning / caution | `#c9954a` | Low-saturation amber for alerts (e.g. "太阳角距过小") |
| Positive / clear | `#5a9e6f` | Low-saturation green for good conditions (e.g. clear sky) |

Color communicates state and hierarchy. No decorative gradients. No glow effects. But do use the *range* within this palette — the difference between 87% white and 55% white is a powerful design tool. The hairline color at 12% opacity creates atmosphere without being decorative.

**Copy style (non-negotiable):**
- Middle-dot `·` separated readout phrases, not sentences
- No exclamation marks, no emoji, no colloquial phrasing
- Astronomy terminology stays professional — no dumbing down
- Locked copy: view one subtitle "Do Look Up——", view one heading "近日可见："

### Typography System

Four-layer font system — each layer has a precise role:

| Layer | Usage | Font | Notes |
|-------|-------|------|-------|
| **Display** | Headings, event names, section titles | Spectral (or Cormorant) | Elegant serif. Use size contrast boldly — a 32px heading next to 13px body text creates drama. |
| **Body / UI labels** | Descriptions, form labels, secondary text | DM Sans | Lightweight, unobtrusive. Does not compete with display or numerics. |
| **Numeric readouts** | Magnitude, distance, angular separation, time, all pure-number values | Geist Mono | Non-negotiable for all numbers. The monospace alignment IS the design. |
| **Chinese text** | All Chinese UI copy and data labels | JingHuaOldSong（京華老宋体）as primary; Noto Serif SC as fallback | JingHuaOldSong has more character and warmth than standard Noto Serif SC. Use it for display-level Chinese text especially. |

Avoid Inter and other pan-use grotesques. Numeric values must always render in Geist Mono.

**Typography as design material:**
The interplay between these four typefaces is the primary decorative system of the product. Specifically:
- **Size jumps create hierarchy and drama:** Don't be afraid of large size ratios. A 28–36px Spectral heading, 13–14px DM Sans body, and 18–22px Geist Mono feature number on the same screen create visual rhythm through contrast alone.
- **Letterspacing as texture:** Section labels in Spectral at small sizes (11–12px) with wide tracking (0.08–0.15em) read as refined, almost ornamental. Geist Mono numbers at default tracking read as precise, scientific.
- **Vertical rhythm matters:** Consistent baseline grids and deliberate line-height choices (1.2 for display, 1.5–1.6 for body, 1.0 for numeric readouts) create the sense of a "typeset page."

### Design Principles

1. **Make data visual.** Numbers and values are the design material. Lay them out as a typesetter would set an almanac. But remember: a well-set almanac page is *beautiful* — it has rhythm, hierarchy, and craft. "Data is the visual" does not mean "only data, nothing else."

2. **No card for everything.** Cards imply containment and separation. Use whitespace, typographic hierarchy, and hairline rules instead. Reserve visible containers for genuinely elevated surfaces (e.g., an expanded observation plan detail, a modal). But "no card" doesn't mean "no structure" — hairline rules, indentation, and spatial grouping should make the information architecture clear.

3. **One accent, used with intention.** The accent color (`#5b8dd9`) marks the most important thing on screen. Don't spray it everywhere. But the muted accent (`rgba(91,141,217, 0.15)`) and hairline accent (`rgba(91,141,217, 0.12)`) can appear more freely to create subtle atmosphere — a hover glow, a section divider, a faint underline on the active state. The palette has range; use it.

4. **Instrument, not publication.** Information density should be high enough to feel like a readout. The user is operating the tool. But a fine instrument is a joy to use — it has considered ergonomics, satisfying tactile feedback (here: hover states, transitions), and an aesthetic that rewards attention.

5. **Motion as state change, with grace.** Transitions confirm that something happened. They should be swift (150–250ms), ease-out, opacity + transform. Support `prefers-reduced-motion`. But within these constraints, transitions should feel *polished* — a card expanding with a coordinated fade of its children, a number updating with a brief cross-dissolve, a table row highlighting on hover with a smooth opacity shift. Motion is not spectacle, but it is not absent either.

6. **Every screen has a moment.** Each view should contain at least one element that rewards a second look — a particularly well-set cluster of data, a hairline that lands perfectly, a hover state that reveals a subtle detail. This is what separates "restrained design" from "unfinished design." The test: if you screenshot the page, does it look like something someone designed on purpose?

### View-Specific Design Guidance

**View Two — Observation Planning Input:**
This view is a form, but it shouldn't look like a generic form. The fields (location, date range, transport mode, equipment) are few enough that each can breathe. Consider: labels positioned to create a visual column, input fields that use the hairline / elevation system rather than heavy borders, a date picker that echoes the ephemeris aesthetic (monospace dates, minimal chrome). The transition from view one (card expanding into this form) is a key design moment — make it feel like the instrument is opening, not like a page is loading.

**View Three — Recommendation Results:**
The most information-dense view and the biggest design opportunity. The observation summary at top should read like a headline data block — large Geist Mono numbers for the key figures (date, magnitude, window), smaller supporting text, the whole thing feeling like the masthead of a data publication. The comparison table is the centerpiece: it should have genuine ephemeris-table beauty. The map and the table have a spatial relationship to design — they're not just stacked, they're composed. When a user clicks a table row to expand the observation plan card, the expansion should feel architectural — content revealing itself with clear hierarchy.

### Accessibility

- Contrast: WCAG AA minimum throughout
- Motion: `prefers-reduced-motion` supported — all animations reduce to instant state change
- No pure red / pure green as sole carriers of critical information (use shape, label, or additional cue)
- No hard requirement for dedicated color-blind mode
- The warning amber (`#c9954a`) and positive green (`#5a9e6f`) are chosen to be distinguishable for common color-vision deficiencies; always pair with text labels
